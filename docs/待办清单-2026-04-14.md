## P0 — 安全与稳定性

### 1. 添加 Helmet 安全中间件

- **位置：** `apps/server/my-blog/src/main.ts`
- **说明：** 生产环境缺少 HTTP 安全头（X-Frame-Options、X-Content-Type-Options、CSP 等）
- **方案：** `pnpm add helmet`，在 `bootstrap()` 中 `app.use(helmet())`

### 2. 添加健康检查端点

- **位置：** `apps/server/my-blog/src/app.controller.ts`
- **说明：** 部署脚本、负载均衡、监控系统都需要 `/health` 端点
- **方案：** 添加 `GET /health` 返回服务状态、数据库连接状态

### 3. 集成 Redis

- **位置：** `apps/server/my-blog/`
- **说明：** `.env.example` 已配置 Redis 连接信息，但代码中完全未使用。当前限流是内存级（重启丢失），热点数据无缓存
- **方案：**
  - 集成 `@nestjs/cache-manager` + `cache-manager-redis-yet`
  - 站点配置、分类/标签列表等热点数据加 Redis 缓存
  - ThrottlerModule 切换到 Redis 存储

## P1 — 功能缺失

### 4. 博客前端 error boundary 和 loading 状态

- **位置：** `apps/blog-web/blog_web/src/app/`
- **说明：** 缺少 `loading.tsx`、`error.tsx`、`not-found.tsx`，用户遇到错误或加载时体验差
- **方案：** 在 `[locale]/` 目录下添加这三个文件

### 5. 动态 sitemap.xml 生成

- **位置：** `apps/blog-web/blog_web/src/app/`
- **说明：** `robots.ts` 引用了 `/sitemap.xml`，但没有实际的 `sitemap.ts` 文件
- **方案：** 创建 `src/app/sitemap.ts`，调用 `getArticleSlugs()` 动态生成

### 6. JWT Refresh Token 机制

- **位置：** `apps/server/my-blog/src/auth/`
- **说明：** 当前 token 过期后只能重新登录，无静默续期能力
- **方案：** 登录时同时返回 refresh_token，添加 `POST /auth/refresh` 端点

### 7. CI/CD 添加测试步骤

- **位置：** `.github/workflows/deploy.yml`
- **说明：** 当前直接 SSH 部署，没有跑测试，有风险把有问题的代码推上线
- **方案：** 在 deploy job 前添加 test job，跑 `pnpm test`

## P3 — 增强功能

### 11. 集成 Giscus 评论系统

- **说明：** 环境变量 `NEXT_PUBLIC_GISCUS_*` 已配置，但博客文章页未集成
- **方案：** 在文章详情页底部添加 Giscus 组件

### 12. 全文搜索优化

- **位置：** `apps/server/my-blog/src/public/public.service.ts` — `search()` 方法
- **说明：** 当前用 `contains` 模糊查询，数据量大时性能差
- **方案：** 使用 PostgreSQL `tsvector` 全文索引，或接入 Elasticsearch/MeiliSearch

### 13. 生产环境结构化日志

- **位置：** `apps/server/my-blog/src/core/logger/`
- **说明：** 当前日志是文本格式，不利于日志平台采集和检索
- **方案：** 生产环境输出 JSON 格式日志（winston / pino）

### 14. 媒体管理 — alt 文本编辑 UI

- **位置：** `apps/admin-web/myapp/src/components/`
- **说明：** 后端全链路支持 alt 字段，但前端上传时没有输入入口
- **方案：** 在 MediaPicker/OssUploader 中添加 alt 输入框，或在媒体管理页支持编辑

### 15. 请求追踪 — Request ID

- **说明：** 当前无分布式追踪能力，排查问题困难
- **方案：** 添加中间件为每个请求生成唯一 ID，写入日志和响应头
