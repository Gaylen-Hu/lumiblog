# 个人博客项目完整架构文档（NestJS + Next.js + 多语言 + CMS模式）

> 目标：建立一个可长期演进、支持多语言、内容可配置、可 SEO 优化的个人博客及内容平台架构。

---

## 一、项目总体架构

### 1. 技术选型

| 层级    | 技术                         | 说明                       |
| ----- | -------------------------- | ------------------------ |
| 后端    | NestJS + TypeScript        | REST API / 可扩展模块化架构      |
| 数据库   | PostgreSQL                 | 支持多语言、JSON、全文搜索          |
| ORM   | Prisma                     | Schema + Migration 管理    |
| 缓存    | Redis                      | 会话、临时数据、热点数据缓存           |
| 前端    | Next.js + React            | SSG / ISR 支持 SEO，中英文路由   |
| 管理端   | React Admin / 自定义组件        | 表单式页面管理，Block 编辑         |
| 存储    | OSS / CDN                  | 媒体资源存储和静态文件分发            |
| 国际化   | i18n（Next.js + 数据库）        | lang 字段 + Translation 表  |
| 反向代理  | Nginx                      | API + 前端路由统一入口，HTTPS/TLS |
| CI/CD | GitHub Actions / GitLab CI | 自动化构建、测试、部署（Phase2/3）    |

### 2. 模块划分

#### 后端模块

* AuthModule（用户 + 权限）
* UserModule（用户信息）
* ContentModule（Page / Post / Block / Theme）
* MediaModule（图片、视频管理）
* SiteConfigModule（站点参数配置）
* SEO / Sitemap 支持
* LogModule（统一日志接口）
* HealthModule（系统监控接口）
* CI/CD 集成接口（预留）

#### 前端模块

* 用户端（博客首页、文章页、分类页、专题页、项目页等）
* 管理端（文章管理、页面管理、主题管理、参数配置、权限管理）
* 公共组件（SEO Head、Layout、PageRenderer、埋点组件）

## 三、前端架构设计

### 1. 用户端

* 路由结构 `/[lang]/[type]/[slug]`
* PageRenderer 根据 Block 渲染页面
* SEO 信息从 ContentTranslation 获取
* SSG / ISR 支持静态化页面和增量更新
* 前端缓存 / Service Worker 可扩展

### 2. 管理端

* 内容管理：文章、页面、项目
* Block 配置：JSON 编辑 + 顺序管理
* 主题管理：选择 Theme 并应用到页面
* 参数管理：SiteConfig
* 用户与权限管理：Role / Permission
* 审计日志（Phase2）预留

### 3. 前端埋点与监控

* track(event, payload) 接口
* 数据可扩展，不影响内容结构
* 错误监控与访问分析接口

---

## 四、SEO、站点地图与国际化设计

* SEO 字段属于 ContentTranslation
* Sitemap 动态生成接口 `/sitemap.xml`，多语言支持
* URL 结构固定 `/[lang]/[type]/[slug]`
* hreflang 标签支持中英文切换
* OpenGraph / Twitter Card 支持
* Next.js 使用 SSG / ISR 静态化页面
* 预留多语言 SEO 扩展和 GEO 定制

---

## 五、架构设计原则

1. 数据模型和 URL 一旦确定，不轻易更改
2. Page / Block / Theme 抽象支持无限扩展
3. 多语言从第一天就支持
4. 权限模型可扩展，多管理员协作
5. 内容与展示层分离，前端渲染器只关心结构与渲染
6. 日志、监控、埋点预留接口即可，Phase 1 不必实现复杂功能
7. 扩展点预留：AI、搜索、个性化推荐、AB 测试
8. CI/CD 流程规划，Phase1 可手动部署，Phase2/3 自动化

## 六、开发与迭代建议

**Phase 0（架构 & 环境）**

* Monorepo + Docker Compose
* NestJS + Prisma + Next.js 初始化
* 数据库建模完成
* Nginx 反向代理 + Redis 基础环境

**Phase 1（MVP）**

* Post / Page 内容模块
* 管理端基本 CRUD + Block 编辑
* 前端 SSG 页面 + SEO + 多语言
* 基础权限与参数设置
* 手动部署 CI/CD 流程

**Phase 2（体验优化）**

* 评论审核、阅读统计、RSS
* 前端 Skeleton / Loading
* 全文搜索 / Redis 热点缓存
* 自动化 CI/CD + 构建测试
* 用户操作审计日志

**Phase 3（平台化 & 高级功能）**

* Page / Project / Landing 扩展
* 高级主题管理 / 可视化布局
* AI 中英文互译 / 内容推荐
* 多作者协作、权限细化
* 灰度发布、自动回滚、CDN 缓存刷新
* 多站点 / GEO 定制

---

## 七、风险点与注意事项

1. 数据模型早期确定，否则返工成本高
2. 国际化和 SEO 架构不可临时加
3. Block type 一旦确定，前端渲染器契约必须稳定
4. 后端动态页面必须和 SSG/ISR 对齐
5. 权限与配置必须在 Phase 1 架构冻结
6. 缓存策略和热点数据处理需要预留接口
7. 日志、监控、运维、备份策略需早期规划
8. CI/CD 流程需预留，Phase2/3 扩展
