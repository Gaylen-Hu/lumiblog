## 项目环境配置

后端服务使用 `.env.[环境]` 文件管理配置：

```
apps/server/my-blog/
├── .env.dev      # 开发环境（已配置）
├── .env.prod     # 生产环境（待配置）
└── .env.example  # 配置模板
```

### 启动命令

```bash
# 进入后端项目目录
cd apps/server/my-blog

# 开发环境
pnpm start:dev

# 生产环境
pnpm start:prod
```

## 注意事项

- `.env.dev` 和 `.env.prod` 文件已添加到 `.gitignore`，不会提交到代码仓库
- 生产环境配置需要单独配置
- 敏感信息请勿直接写入代码中，统一使用环境变量
