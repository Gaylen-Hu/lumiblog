# 文章评分体系（百分制）

> 镜微审校专用。每篇文章必须经过此评分体系，70 分以下打回重写。

---

## 零、100 分的文章长什么样

不存在完美的文章，但 90+ 的文章有几个共同特征：

读完之后你会想转发给某个具体的人，并且附上一句自己的评论。开头让你停下了正在刷的手，中间有一两个地方让你"嗯"了一声，结尾没有说教但你确实想了一会儿。里面有一句话你想截图，不是因为它"正确"，而是因为它说出了你一直想说但没组织好语言的东西。文章里有一两处让你笑了，或者至少嘴角动了一下。读完之后你记得住这篇文章讲了什么，而不只是"好像看过一篇关于 XX 的文章"。

对标范文：AI炼金术《当产能不再稀缺，什么才稀缺？》——那篇用星巴克咖啡师开头，层层剥洋葱，结尾不鸡汤但让人沉默。

## 一、评分总览

| 维度 | 满分 | 说明 |
|------|------|------|
| A. 选题价值 | 15 | 这篇文章值不值得写 |
| B. 开头钩子 | 10 | 前三行能不能让人停下拇指 |
| C. 结构与流动 | 12 | 骨架撑得住，但不能像骨架标本 |
| D. 信息密度 | 13 | 每段是否都有新信息 |
| E. 金句与观点 | 10 | 有没有让人想截图的句子 |
| F. 文风与节奏 | 12 | 读起来像人写的还是机器拼的 |
| G. 趣味与人味 | 8 | 有没有让人笑、让人意外、让人觉得"这是个活人写的" |
| H. 事实准确性 | 8 | 数据、引用、版本号是否靠谱 |
| I. 分发物料 | 8 | 标题、文案、话题能不能带来点击 |
| J. 排版与可读性 | 4 | 手机上读起来舒不舒服 |
| **总计** | **100** | |

**评级标准：**
- 90-100：可直接发布，标杆级
- 80-89：小改后发布
- 70-79：需要修改 🟡 项后重新提交
- 60-69：打回重写，结构或选题有问题
- 60 以下：废稿，重新选题

## 二、各维度详细评分标准

### A. 选题价值（15分）

| 分数 | 标准 |
|------|------|
| 13-15 | 切中行业真实痛点，有认知差或信息差，读者看完能改变行动 |
| 10-12 | 话题有价值，角度有一定差异化，但不是非读不可 |
| 7-9 | 话题正确但角度平庸，和同类文章没有明显区别 |
| 4-6 | 入门级科普或信息搬运，没有独立观点 |
| 0-3 | 选题过时、与读者无关、或和近期已发文章严重撞车 |

**扣分项：**
- 同一批次两篇文章素材来源重叠超过 40%：直接扣 5 分
- 选题无法在 30 秒内说清"为什么值得写"：扣 3 分
- 目标读者不明确（谁都能读 = 谁都不想读）：扣 2 分

### B. 开头钩子（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 前三行制造了强烈的好奇心、冲突感或画面感，让人必须往下读 |
| 7-8 | 开头有吸引力，但钩子不够锐利，可以更好 |
| 5-6 | 开头平淡，没有明显的钩子，但也没有劝退 |
| 3-4 | 开头是背景介绍或概念解释，读者可能在第二段划走 |
| 0-2 | "随着AI的发展..." 式开头，或直接从定义开始 |

**加分项（每项 +1，上限 10）：**
- 用了具体场景或画面开头
- 用了反直觉的数据或判断
- 用了真实人物的一句话作为引子
- 开头本身就是一个可传播的观点

**扣分项：**
- 开头超过 3 行才进入正题：扣 2 分
- 使用禁用句式开头（"在当今..." / "随着..."）：直接 0 分

### C. 结构与流动（12分）

> 注意：这个维度评估的不是"逻辑有多严密"，而是"读起来是否自然流畅"。
> 过度工整的结构（每段一个论点、严格递进、像论文大纲）反而会扣分，因为那是 AI 最擅长的写法。
> 好的结构像河流——有方向，但允许拐弯。

| 分数 | 标准 |
|------|------|
| 10-12 | 有清晰的方向感，但不死板。有主线也有支流，读者跟着走不会迷路，偶尔的"跑题"反而让人觉得有趣 |
| 7-9 | 结构清晰，过渡自然，但太工整了——像 PPT 大纲，缺少意外感 |
| 4-6 | 结构基本成立，但有逻辑跳跃，或者太松散 |
| 0-3 | 没有方向感，信息堆砌，前后矛盾 |

**检查项：**
- 开头提出的问题，结尾是否回答了？（未回答扣 3 分）
- 论据是否支撑论点？（偷换概念扣 3 分）
- 是否使用了刻板连接词？（"综上所述""总而言之"每处扣 1 分）
- 结尾是否有空洞升华？（"这不仅...也预示着..."扣 2 分）
- 全文是否过度工整？（每段都是"论点→论据→小结"的三段式，扣 2 分——这是 AI 写作的典型特征）

**加分项（每项 +1）：**
- 有一处有意的"跑题"或闲笔，但读完发现它其实服务于主题
- 有一处意外的转折或视角切换，让读者"没想到会聊到这个"
- 结尾不是总结，而是一个新的问题或画面

### D. 信息密度（13分）

| 分数 | 标准 |
|------|------|
| 11-13 | 每段都有新事实、新数据或新观点，没有一句废话 |
| 8-10 | 大部分段落有信息量，但有 1-2 段可以删掉不影响理解 |
| 5-7 | 信息量中等，有明显的注水段落 |
| 3-4 | 信息稀薄，大量段落在说"正确的废话" |
| 0-2 | 通篇空洞，没有具体事实支撑 |

**逐段检查法：**
对每一段问："删掉这段，文章缺了什么？"
- 答案是"没缺"的段落数 × 1 分 = 扣分数
- 同一个观点换措辞重复：每次扣 1 分

### E. 金句与观点（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 有 2-3 句让人想截图发朋友圈的判断句，观点锐利且有理有据 |
| 7-8 | 有 1-2 句不错的金句，但锐度可以更强 |
| 5-6 | 有观点但表达平淡，没有让人记住的句子 |
| 3-4 | 观点模糊，骑墙 |
| 0-2 | 没有独立观点，纯信息搬运 |

**金句的"人味检验法"：**
想象你在一个技术人的饭局上，有人说了这句话——
- 桌上的人会不会停下筷子？会 → 好金句
- 会不会有人说"等等，你再说一遍"？会 → 很好的金句
- 会不会有人当场反驳，然后吵起来？会 → 最好的金句
- 大家礼貌点头然后继续吃？→ 不算金句，只是正确的废话

**价值观合规底线：**
- 涉及歧视、人身攻击、政策敏感：直接扣 3 分
- 判断标准：这句话被截图传播，Boss 会不会被骂？

### F. 文风与节奏（12分）

**F1. 去 AI 味（6分）**

| 分数 | 标准 |
|------|------|
| 5-6 | 读起来完全像人写的，有个人风格 |
| 3-4 | 基本没有 AI 味，但某些段落偏"报告体" |
| 1-2 | 有明显 AI 痕迹 |
| 0 | 通篇 AI 味 |

逐项扫描（每发现一处扣 0.5 分）：
- 禁用词汇和禁用句式
- 三项并列列举
- 连续三句长度相同
- "进行 + 名词"组合
- 同义词循环

**F2. 节奏感（6分）**

| 分数 | 标准 |
|------|------|
| 5-6 | 长短句交替自然，有呼吸感 |
| 3-4 | 节奏基本可以，某些段落偏平 |
| 1-2 | 节奏单调 |
| 0 | 像说明书 |

检查项：
- 段落不超过 5 行
- 有对话感（"你""我"）
- 段落结尾多样化
- 有留白和断句

### G. 趣味与人味（8分）🆕

> 这是区分"合格的技术文章"和"让人想追更的技术文章"的关键维度。
> 半佛仙人的文章逻辑未必最严密，但你就是想看，因为他有趣。

| 分数 | 标准 |
|------|------|
| 7-8 | 至少有 2 处让读者笑了或意外了，有个人经历或独特视角穿插，读完觉得"这个作者挺有意思" |
| 5-6 | 有 1 处幽默或意外，整体不无聊但也没有惊喜 |
| 3-4 | 全文正经到底，信息准确但读起来像在看说明书 |
| 0-2 | 枯燥，读到一半想划走 |

**趣味来源（不限于此，任何让人觉得"这不像 AI 写的"的元素都算）：**
- 黑色幽默或毒舌（"每一段废话都是对读者时间的谋财害命"）
- 意外的类比（"用星巴克咖啡师解释产能过剩"）
- 个人经历或真实感受（"我第一次看到这个数据的时候愣了三秒"）
- 自嘲或承认不确定（"说实话我也没完全想明白"）
- 画面感强的场景描写（不是抽象论述，是让人"看到"）
- 反转或意外的结论（读者以为你要说 A，结果你说了 B）
- 对某个现象的吐槽（不是中立报道，是有态度的吐槽）

**扣分项：**
- 全文没有任何一处让人嘴角动一下的地方：上限 3 分
- 幽默生硬或尬（强行抖机灵比不幽默更糟）：扣 2 分

**加分项：**
- 有一处幽默让人想截图分享（和金句不同，金句是观点，幽默是态度）：+1 分

### H. 事实准确性（8分）

| 分数 | 标准 |
|------|------|
| 7-8 | 所有数据有出处，版本号准确，附带素材来源清单 |
| 5-6 | 大部分准确，有 1-2 处需标注"待核实"，来源清单基本完整 |
| 3-4 | 事实模糊或来源清单缺失 |
| 1-2 | 有事实错误 |
| 0 | 编造引用或虚假数据 |

**前置要求——素材来源清单（和光必须提供）：**
和光提交稿件时，必须在文末附带素材来源清单：
```
**参考来源：**
- [来源标题](链接或文件路径) — 用于第X段的XX论点
```
未提供来源清单：H 项上限 4 分。

### I. 分发物料（8分）

| 分数 | 标准 |
|------|------|
| 7-8 | 5 个标题中至少 2 个让人想点，3 条社群文案可直接发群，2 个互动话题能引发讨论 |
| 5-6 | 物料齐全，标题有 1 个不错的 |
| 3-4 | 物料齐全但平庸 |
| 1-2 | 物料不全或质量差 |
| 0 | 没有配套物料 |

**标题的"人味检验法"：**
想象这个标题出现在你的微信公众号列表里，上面是"差评"的文章，下面是"36kr"的文章——
- 你会点吗？不会 → 标题不及格
- 你会犹豫一下？→ 及格
- 你会立刻点？→ 好标题
- 你会点完之后还想看看评论区？→ 很好的标题

**标题检查：**
- 5 个标题是否覆盖悬念、利益、故事、争议四种类型？
- 标题是否在 20 字以内？
- 标题有没有制造"信息缺口"？（让人觉得"不点进去就不知道答案"）

**联动规则：**
- B ≥ 8 但 I ≤ 4：标记"好文章但出不去"，I 必改
- I ≥ 6 但 B ≤ 5：标记"标题党风险"，B 必改

### J. 排版与可读性（4分）

| 分数 | 标准 |
|------|------|
| 4 | Markdown 规范，配图占位符合理，手机友好 |
| 2-3 | 格式基本正确，有小问题 |
| 0-1 | 格式混乱 |

## 四、特殊规则

### 同批次去重检查
同一天产出的文章之间，素材来源重叠度不得超过 40%。
超过 40%：后写的那篇 A 项直接扣 5 分。

### 禁用词一票否决
- 出现 1 处：F1 扣 1 分
- 出现 3 处以上：F1 直接 0 分
- 开头使用禁用句式：B 直接 0 分

### 编造引用零容忍
编造引用、虚假数据或伪造人物发言：H 直接 0 分，全文降为废稿。
