## 选题B 大纲

topic: 代码保质期只剩三个月——"为六个月后的模型构建"到底是什么意思
audience: AI产品经理、技术创业者、架构师
angle: 从Boris Cherny的"潜在需求"和Rich Sutton的"苦涩的教训"出发，推导AI时代的产品决策框架
structure_pattern: 层层剥洋葱法

outline:
  - section: "Boris的CLAUDE.md只有两行"
    key_point: Claude Code负责人自己只用两行配置，建议"太长就删掉重写"。这不是偷懒，是对模型演进速度的敬畏。80%的代码库是近几个月重写的。脚手架能带来的10-20%性能提升，新模型一出就被抹平。
    reference: /root/qqopenclawlgo/articles/Gino Notes/Claude Code 一周年：Boris Cherny 访谈中最值得开发者深思的 5 件事.md
  - section: "苦涩的教训：通用模型总是赢"
    key_point: 引入Rich Sutton的经典论文《The Bitter Lesson》。每一次特定领域的精细工程优化，最终都被更通用的模型覆盖。Claude Code团队的工程决策框架：脚手架 vs 等模型，怎么选？
    reference: /root/qqopenclawlgo/articles/Gino Notes/Claude Code 一周年：Boris Cherny 访谈中最值得开发者深思的 5 件事.md
  - section: "一个不舒服的推论：前六个月PMF可能极差"
    key_point: "为六个月后的模型构建"意味着产品刚上线时体验可能很差。Boris直言"这会让人很不舒服"。但这恰恰是护城河——当模型追上你的设计，你就能立刻起飞。
    reference: /root/qqopenclawlgo/articles/Gino Notes/Claude Code 一周年：Boris Cherny 访谈中最值得开发者深思的 5 件事.md
  - section: "决策框架：什么时候该建，什么时候该等"
    key_point: 可操作的判断标准：①如果这个投入只在"当前模型不进步"时才有价值→不建；②如果能随模型变强而变得更有价值→建；③如果六个月后模型自己就能做→等。
    reference: /root/qqopenclawlgo/articles/AI炼金术/当产能不再稀缺，什么才稀缺？.md
  - section: "对普通开发者的启示"
    key_point: "保持初学者心态"比经验更值钱。资深工程师的思维有时反而成为障碍。你该投资的是"知道让AI做什么"的能力，而不是"自己做得更好"的能力。
    reference: /root/qqopenclawlgo/articles/AI前线/智能体 AI 模式强化软件工程规范性.md
