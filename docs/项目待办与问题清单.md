## 一、核心功能缺失

### 1.1 文章与分类/标签关联（高优先级）

**问题描述：** 数据库 schema 中 Article 表缺少与 Category、Tag 的关联关系。

**当前状态：**
- Prisma schema 中 Article 模型没有 `categoryId` 和 tags 关联字段
- 后端 `ArticleService` 的 create/update 方法未处理分类和标签
- 前端管理后台已实现分类/标签选择器，但数据无法持久化

**需要修改：**
```prisma
// apps/server/my-blog/prisma/schema.prisma
model Article {
  // ... 现有字段
  categoryId String?
  category   Category? @relation(fields: [categoryId], references: [id])
  tags       ArticleTag[]
}

model ArticleTag {
  id        String   @id @default(uuid())
  articleId String
  tagId     String
  article   Article  @relation(fields: [articleId], references: [id], onDelete: Cascade)
  tag       Tag      @relation(fields: [tagId], references: [id], onDelete: Cascade)
  
  @@unique([articleId, tagId])
  @@map("article_tags")
}

model Tag {
  // ... 现有字段
  articles ArticleTag[]
}
```

**影响范围：**
- `apps/server/my-blog/src/article/article.service.ts` - 需要处理关联
- `apps/server/my-blog/src/article/dto/*.ts` - 需要添加关联字段
- `apps/server/my-blog/src/public/public.service.ts` - 需要从数据库读取关联

## 二、数据模型缺失

### 2.1 项目（Project）模型未实现

**问题描述：** 数据库中没有 Project 表，无法管理项目/作品集。

**当前状态：**
- Prisma schema 中没有 Project 模型
- 后端 `PublicService` 中项目数据为 Mock
- 管理后台没有项目管理页面

**需要添加：**
```prisma
model Project {
  id          String   @id @default(uuid())
  title       String
  description String   @db.Text
  techStack   String[] // PostgreSQL 数组类型
  coverImage  String?
  link        String?
  githubUrl   String?
  featured    Boolean  @default(false)
  order       Int      @default(0)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@map("projects")
}
```
